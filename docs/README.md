## Contributing to Documentation

When adding new features or modifying existing behavior:

1. Update relevant documentation files
2. Add new documents if covering a new major feature
3. Keep code examples up to date
4. Document edge cases and security considerations
5. Update this README with new document links

## Documentation Standards

- Use clear, descriptive headings
- Include code examples where relevant
- Document both expected and edge case behaviors
- Add diagrams for complex flows (when applicable)
- Keep changelog updated
- Cross-reference related documents

## Questions?

If documentation is unclear or missing information, please:
1. Check the referenced source code
2. Review test files for examples
3. Open an issue describing what needs clarification
